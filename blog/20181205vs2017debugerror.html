VS2017调试Web程序错误（无法生成调试程序包）

2018-12-05 / sagaii

﻿

一、 原由

最近修改一个小程序(Asp.net MVC架构)，在自带的iisexpress上调试。

  1. 一直报错误： 无法访问请求的页面,因为该页的相关配置数据无效。
  2. 查询 iisexpress配置文件夹 %userprofile%\documents\iisexpress 当中的配置文件，发现代码没有创建相应的站点配置。
  3. 在%userprofile%\documents\my web sites\文件夹下站点也缺少相应站点网页（含程序）。

二、 分析

  1. 有可能是权限问题，导致iisexpress无法访问上述文件夹。 因此，配置了权限，结果失效。
  2. 还有可能是因为我配置了文件夹的符号链接（mklink命令创建)，导致vs2017访问符号链接文件夹失败。
  3. 清理了asp.net 项目工程，无法解决问题。
  4. 考虑可能是vs2017没有正常发布调试程序。

三、 解决

根据以上判断，搜索后，确定方案，删除工程文件夹当中的.vs文件夹，重新生成文件即可。

因为： .vs目录是用来存储当前用户在解决方案中的工作配置，具体包括VS关闭前最后的窗口布局、最后打开的选项卡/操作记录/文件文档、某些自定义配置/开发环境、调试断点等这类设置信息和状态。这样每当用户关闭解决方案后再重新打开，就能继续之前的工作状态。

删除之后，vs会重新创建所有初始化配置参数，包括向iisexpress的配置文件添加项目配置信息和向my web sites文件夹创建初始程序文件。
